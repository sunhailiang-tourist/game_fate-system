# 因果修真世界 - 随机事件系统

## 一、系统概述

### 1.1 设计目标
随机事件系统旨在实现：
- **真实世界模拟**：模拟物理世界的各种随机事件
- **命运无常体验**：事件结果具有不确定性
- **玩家影响**：事件对玩家造成伤害、属性变化、命运转折
- **世界互动**：事件与世界状态相互影响

### 1.2 核心机制（事件驱动，非任务链）

```
状态变化 → 事件检测 → 概率判定(hazard) → 结果计算 → 玩家影响 → 世界反馈
```

人生事件由饥饿、境界、环境、关系等 **状态变化** 触发，不使用任务脚本链。

### 1.3 风险曲线死亡（Hazard 模型）

死亡应 **可追溯原因**，避免暗箱暴毙：

```
hazard(t) = base_risk × env_factor × merit_modifier × imprint_modifier
```

- 每项风险（饥饿、中毒、渡劫、战斗）独立累积 hazard
- 死亡时记录 **死因链**（如：饥饿→虚弱→遇袭→阵亡）
- UI 以叙事反馈，不展示精确概率数值

### 1.4 黑天鹅事件

极低概率、高影响的改命事件（凡人开挂、逆天机缘）：

| 约束 | 说明 |
|------|------|
| 基础概率 | ≤ 0.01% / 检测周期 |
| 冷却 | 同账号每世最多 1 次同类型 |
| 防刷穿 | 不可通过重复行为堆叠概率 |

## 二、事件分类体系

### 2.1 事件类型

| 事件类型 | 触发场景 | 频率 | 影响范围 |
|----------|----------|------|----------|
| **自然事件** | 天气、环境变化 | 高 | 区域/世界 |
| **意外事件** | 随机事故 | 中 | 单个玩家 |
| **机缘事件** | 奇遇、宝藏 | 低 | 单个玩家 |
| **灾难事件** | 自然灾害 | 低 | 区域 |
| **社交事件** | NPC互动 | 中 | 单个玩家 |
| **战斗事件** | 遭遇敌人 | 中 | 单个玩家 |
| **修炼事件** | 修炼中随机事件 | 中 | 单个玩家 |
| **命运转折** | 重大人生事件 | 极低 | 单个玩家 |

### 2.2 事件等级

| 等级 | 名称 | 影响程度 | 触发概率 |
|------|------|----------|----------|
| 1 | 微小 | 轻微影响 | 高 |
| 2 | 普通 | 中等影响 | 中 |
| 3 | 重大 | 显著影响 | 低 |
| 4 | 命运 | 决定性影响 | 极低 |

## 五、机缘事件系统

### 5.1 奇遇事件

| 事件 | 触发条件 | 奖励类型 | 稀有度 |
|------|----------|----------|--------|
| **发现宝藏** | 探索秘境 | 金币/道具 | 稀有 |
| **遇到高人** | 随机触发 | 功法/指点 | 传说 |
| **获得传承** | 机缘巧合 | 特殊能力 | 传说 |
| **天降机缘** | 高功德 | 随机奖励 | 稀有 |
| **灵兽认主** | 高魅力 | 获得宠物 | 传说 |

### 5.2 机缘概率

```
机缘概率 = 基础概率 × 功德修正 × 幸运值修正 × 随机因子

基础概率: 0.1%
功德修正: 每1000功德 +10%
幸运值修正: 每10点幸运 +5%
随机因子: 0.5-1.5
```

## 八、事件对玩家的影响

### 8.1 伤害类型

| 伤害类型 | 说明 | 恢复方式 |
|----------|------|----------|
| **物理伤害** | 直接生命值损失 | 自然恢复/疗伤 |
| **精神伤害** | 精神力损失 | 冥想/丹药 |
| **中毒伤害** | 持续伤害 | 解毒药 |
| **诅咒伤害** | 负面状态 | 解除诅咒 |
| **属性伤害** | 属性降低 | 恢复药剂 |

### 8.2 属性影响公式

```
属性变化 = 基础值 × 事件系数 × 玩家抗性

事件系数: 根据事件等级(-50%~+50%)
玩家抗性: 装备/功法提供的抗性(0~100%)
```

### 8.3 状态影响

| 状态类型 | 效果 | 持续时间 |
|----------|------|----------|
| **中毒** | 持续掉血 | 5-30分钟 |
| **虚弱** | 属性-20% | 10分钟 |
| **兴奋** | 属性+10% | 5分钟 |
| **恐惧** | 逃跑概率增加 | 3分钟 |
| **祝福** | 全属性+10% | 1小时 |

## 九、事件触发机制

### 9.1 触发条件

| 条件类型 | 说明 | 示例 |
|----------|------|------|
| **时间触发** | 特定时间触发 | 午夜、节气 |
| **位置触发** | 特定区域触发 | 秘境、危险区域 |
| **行为触发** | 特定行为触发 | 修炼、战斗 |
| **状态触发** | 特定状态触发 | 低血量、高功德 |
| **随机触发** | 随机概率触发 | 日常事件 |

### 9.2 事件检测流程

```
玩家行动 → 环境检测 → 概率判定 → 事件生成 → 结果执行
    ↓          ↓          ↓          ↓          ↓
 行为记录   状态扫描   随机计算   事件选择   效果应用
```

### 9.3 事件冷却机制

| 事件类型 | 冷却时间 | 说明 |
|----------|----------|------|
| 微小事件 | 无冷却 | 可连续触发 |
| 普通事件 | 5分钟 | 同一事件 |
| 重大事件 | 1小时 | 同一事件 |
| 命运事件 | 1轮回 | 同一类型 |
