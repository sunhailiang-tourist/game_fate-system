# 因果修真世界 - 选胎界面规格

> **用途**：死亡后投胎 UI/UX 与交互规则。  
> **版本**：v1.0 | 2026-06-12

---

## 一、流程入口

```
死亡结算完成 → [可选] 魂修短阶段 → 选胎界面 → 确认 → 新一世加载
```

**前置条件**：
- `active_life_id` 对应实体已 `dead`
- 投胎池 `pool` 已由服务端生成
- 单命制：界面打开期间不得有其他活跃生命

---

## 二、魂修短阶段（可选）

| 元素 | 说明 |
|------|------|
| 时长 | 30s~2min 现实时间，可跳过 |
| 内容 | 冥界氛围、本世关键抉择闪回（只读） |
| 交互 | 「继续前行」进入选胎；无战斗、无采集 |
| 自杀 | 闪回更少，氛围更压抑；仍保留印记提示（叙事化） |

---

## 三、选胎界面布局

```
┌────────────────────────────────────────────┐
│  轮回审判（叙事文案，非数值）                  │
│  「此生功业已结算，前路在汝一念之间…」          │
├────────────────────────────────────────────┤
│  [人道·平民]  [人道·商贾]  [畜生道·灵兽]      │
│  [植物道·灵草]  [灰显：仙界候补]  ...          │
├────────────────────────────────────────────┤
│  选项详情：物种 / 身份档位 / 出生地品质描述      │
│  （不显示功德数值、判定值）                    │
├────────────────────────────────────────────┤
│  [轮回档案]          [确认投胎]  [暂不]       │
└────────────────────────────────────────────┘
```

---

## 四、选项状态

| 状态 | 表现 | 交互 |
|------|------|------|
| **可选** | 正常卡片 | 可点击选中 |
| **灰显** | 锁定图标 + 模糊原因（「功业未逮」「命数未至」） | 不可选 |
| **推荐** | 轻微光晕（基于池内默认，非强制） | 可改选 |

> 灰显原因 **不得** 写「需要功德 5000」等精确数值。

---

## 五、交互规则

1. 玩家必须 **显式确认** 一次投胎选择
2. 确认前可切换池内任意可选卡片
3. 确认后 `POST /reincarnation/choose`，服务端二次校验
4. 具体出生地、家族、初始属性在档位内 **随机生成**（加载画面展示）
5. 「轮回档案」侧栏：只读多世历史，不可操控旧生命

---

## 六、自杀死亡差异

| 项 | 正常死亡 | 自杀 |
|----|----------|------|
| 审判文案 | 中性 | 带警示语气 |
| 池子 | 按本世功德 | 已 -500 后的缩减池 |
| 记忆 | 可有碎片提示 | 明确「记忆散尽」 |
| 印记 | 继承 | **保留**（档案中可见印记图标） |

---

## 七、恶意杀害

- 无「被害补偿」专属 UI 或优渥选项高亮
- 与正常死亡同一选胎流程，池子仅按死者功德计算

---

## 八、错误与边界

| 场景 | 处理 |
|------|------|
| 选胎超时（如 30min） | 池内随机一档可选项（非全随机六道） |
| 网络断开 | 保留 pool 快照，重连恢复 |
| 重复提交 | 幂等：已选则返回当前新生命 |

---

## 九、数据依赖

- `GET /reincarnation/pool`
- `POST /reincarnation/choose`
- `GET /reincarnation/archive`
- `SoulArchive`, `ReincarnationPool` 表（见数据结构文档）

---

**适用范围**：前端、UX、后端
